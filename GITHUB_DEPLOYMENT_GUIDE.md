# GitHub Pages Deployment Guide for Group Portfolio Website

Follow these steps to publish your website at: https://yourusername.github.io/group-portfolio/

---

## Step 1: Create a GitHub Account

1. Go to [github.com](https://github.com)
2. Click **"Sign up"**
3. Enter your email, create a password, and choose a username
4. Complete the verification
5. Choose the free plan (free)

---

## Step 2: Create a New Repository

1. After logging in, click the **"+"** icon in the top-right corner
2. Select **"New repository"**
3. Fill in the details:
   - **Repository name**: `group-portfolio`
   - **Description**: Group Portfolio Website
   - **Visibility**: Select **"Public"**
   - âœ… Check **"Add a README file"**
4. Click **"Create repository"**

---

## Step 3: Upload Your Website Files

1. In your new repository, click the **"Add file"** button (near "Go to file")
2. Select **"Upload files"**
3. Drag and drop ALL of these files from your project folder:
   - `portfolio.html`
   - `style.css`
   - `script.js`
   - `pic1.jpg`
   - `pic 2.jpg`
   - `pic 3.jpg`
   - `pic 4.jpg`
   - `pic 5.jpg`
   - `pic 6.jpg`
   - `pic 7.jpg`
   - `pic 8.jpg`
   - `pic 9.jpg`
   - `pic 10.jpg`
4. Click **"Commit changes"** (green button)

---

## Step 4: Enable GitHub Pages

1. In your repository, click the **"Settings"** tab (right side, near top)
2. In the left sidebar, click **"Pages"**
3. Under **"Build and deployment"**:
   - **Source**: Select **"Deploy from a branch"**
4. Under **"Branch"**:
   - **Branch**: Select **"main"** (or "master")
   - **Folder**: Select **"/(root)"**
5. Click **"Save"**

---

## Step 5: View Your Published Website

1. Wait about 1-2 minutes for GitHub to deploy
2. Refresh the Pages settings page
3. Look for the message: "Your site is live at https://yourusername.github.io/group-portfolio/"
4. Click the link to view your website!

---

## Important Notes

- Replace `yourusername` with your actual GitHub username
- The URL format will be: `https://yourusername.github.io/group-portfolio/`
- Any future changes you push to the repository will automatically update the live site
- Make sure all image filenames match exactly (including spaces)

---

## Troubleshooting

If images don't load:
- Check that all image files were uploaded
- Verify filenames in portfolio.html match the uploaded files exactly
- Image names with spaces need to be referenced with spaces in HTML (e.g., `src="pic 2.jpg"`)

---

Done! Your Group Portfolio website will be live on the internet for everyone to see!
